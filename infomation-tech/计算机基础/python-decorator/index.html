  装饰器的使用显得很Pythonic，在源码中装饰器也很常见。用一个数学例子作为类比，理解一下闭包和装饰器。

装饰器是什么？

装饰器本质上是一个闭包，它接受一个函数作为参数，并返回函数。可以不改变代码情况下，动态修改函数功能。

  有了装饰器，我们就可以抽离出大量与函数功能本身无关的雷同代码并继续重用。在说明装饰器原理前，还要说另一个概念：闭包。

闭包

  闭包就是能够读取其他函数内部变量的函数。

def outer():
    x = 1
    def inner():
        print(x)
    return inner

f = outer()
f() # 1

  x是外部函数outer的local variable，当outer调用结束后，按理说，变量x的生命周期就结束了。但是inner能够正常的读取到x，相当于变量的生命周期变长了。这是因为Python支持了闭包特性。

  原理就是函数内部__closure__属性记录了enclosing scope中的non_local variable。详细原理可见博客Python Closure，从汇编的角度剖析了闭包的原理。

装饰器

  Python的装饰器就是对闭包的一种应用，先看一个数学例子。

装饰器代码

  将上述过程用代码实现：

# 闭包
def decorator(func):
    def wrapper(*args, **kw):
        return 3*func(*args,**kw)-2
    return wrapper

# 原函数
def f(x):
    return 2*x+1

@decorator
def f(x):
    return 2*x+1

# 现在的f(x)已经是新函数了
# @语法糖的作用是重命名：f = decorator(f) 
print(f(1))  # ret:7=6*1+1

  可以发现装饰器函数就是对原函数包了一层，额外添加一些功能(或者说是闭包)。闭包已经实现了新函数的功能，通过@语法糖可以省略f=F(f)的显式声明，现在使用原函数的名字f(x)f(x)f(x)就可以实现新函数的作用。

怎么写装饰器？

  博客详解Python的装饰器关于这方面内容就很详细完整，这里的顺序和参考博客一样。

普通装饰器

  通过装饰器，增加函数功能:运行时，打印出函数名字和函数的内存地址。

def show(func):
    def wrapper(*args, **kwargs):
        print(func.__name__, id(func))
        return func(*args, **kwargs)
    return wrapper

@show
def add(a, b):
    print(a+b)

add(2,3)
add 336812640
5

  不难看出，实际上现在的add函数通过装饰器的修饰，实际上指向装饰器函数show的返回函数wrapper。wrapper中的逻辑就是新的函数的逻辑，这里不仅实现了原函数add的功能，还额外添加了打印函数名字和id的功能。

  可见，想要知道一个装饰器的作用应该查看装饰器函数最内层的函数的逻辑。

带参数的装饰器

  现在还想在运行函数时，如果函数所在的内存大于指定数字就报警。那么，我们需要给装饰器传一个参数来传递指定数字。

  可是在上述的代码中，装饰器函数所实现的闭包必须要以函数作为参数，不能改动。通过之前经验，可以给装饰器函数再包一层就行了，这次最外层的函数就是为了传递参数。

def show(num):
    def outer_wrapper(func):
        def wrapper(*args, **kwargs):
            print(func.__name__, id(func))
            if id(func)>num:
                raise ValueError("报警了")
            return func(*args, **kwargs)
        return wrapper
    return outer_wrapper

@show(30000000)
def add(a, b):
    print(a+b)

类装饰器

  有的时候一些源码中使用的是类当装饰器函数，类怎么当函数？其实一个类只要实现了__call__方法，就能像函数一样被调用。

class Test:
    def __init__(self, num):
        self.num = num
    def __call__(self, b):    
        print(self.num + b)

t1 = Test(5)
t1(3)
8

  t1是Test类的实例化对象，由于Test实现了__call__方法，所以t1可以像一个函数一样被调用。那么，通过类来实现装饰器。

class show:
    def __init__(self, func):
        self.func = func
    def __call__(self, *args, **kwargs):
        print(self.func.__name__, id(self.func))
        return self.func(*args,**kwargs)
        
@show
def add(a, b):
    print(a+b)

add(3,2)
add 337672936
5

  类比于函数作装饰器，类中的__init__函数相当于外层函数，而__call__相当于能够添加函数功能的内层函数。如果要实现带参数的装饰器的话，修改如下：

class show:
    def __init__(self, num):
        self.num = num
        
    def __call__(self, func):
        def wrapper(*args,**kwargs):
            print(func.__name__, id(func))
            if id(func) > self.num:
                raise ValueError("报警了")
            return func(*args,**kwargs)
        return wrapper
        
@show(3000000)
def add(a, b):
    print(a+b)

装饰器注意点

  参考博客详解Python的装饰器中提到的错误的函数签名和文档值得令人注意。通过装饰器修饰后，原来的函数__name__实际上已经改变了。

add.__name__
Out[34]: 'wrapper'

这点其实正好印证了上面的说法。如果就想要add通过装饰器后和原来完全一样怎么办？使用标准库里的functools.wraps:

def show(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        print(func.__name__, id(func))
        return func(*args, **kwargs)
    return wrapper
@show
def add(a, b):
    print(a+b)

装饰器的应用？

  工作中，装饰器有没有作用呢？目前来说，我使用装饰器大多是一些简单的功能，具体记录如下：

函数计时器

  有些函数需要在有限时间内完成。比如，在跑spark任务的时候，发现有的时候，会卡在某些stage或者其他地方，导致任务很长时间都跑不完。在原来函数的基础上加上计时器来及时中断这些任务。

'''使用Linux alarm信号来使限制函数运行时间的装饰器'''
import signal
import time

def timeout(seconds):
    def wrapper(func):
        def timeout_callback(signum, frame):
            raise TimeoutError("Run %s time out, limited time is %d s." % (func, seconds))

        def inner_wrapper(*args, **kwargs):
            signal.signal(signal.SIGALRM, timeout_callback)  # unix only
            signal.alarm(seconds)  # set alarm
            r = func(*args, **kwargs)
            signal.alarm(0)  # close alarm
            return r

        return inner_wrapper

    return wrapper


@timeout(5)
def test():
    print("start")
    time.sleep(6)
    print("end")


if __name__ == '__main__':
    try:
        c = test()
    except TimeoutError:
        print("function is timeout")
        pass

函数定时器

  有些函数需要在指定时间进行。线上机器学习模型是通过小批量不断更新的，每隔两小时跑一次evaluation，而每天12点定时删除一些旧的模型数据。

import datetime

def cron(interval, rem):
    def wrapper(func):
        def inner_wrapper(*args, **kwargs):
            cur = datetime.datetime.now().strftime("%H:%M")
            cur_hour, cur_min = cur.split(":")
            if int(cur_hour) % interval == rem:
                return func(*args, **kwargs)
            else:
                return

        return inner_wrapper

    return wrapper

# 每天12点跑
@cron(24, 12)
def task1():
    pass

# 每隔2小时跑
@cron(2, 0)
def task2():
    pass

预激生成器

  python协程可以通过生成器实现。例如下面的consumer协程需要预先跑到yield的地方才能使用，通过装饰器来预激生成器。

def coroutine(func):
    """预激生成器"""
    def wrapper(*args, **kwargs):
        gen = func(*args, **kwargs)
        next(gen)
        return gen

    return wrapper

@coroutine
def consumer(args):
    while True:
        production = yield
        do_consume(production, args)
        
def producer(consumer, args):
    while True:
        product = do_produce(args)
        consumer.send(product)
        
c = consumer(arg_dict)
producer(c, arg_dict)

函数注册

  将函数注册到指定的字典中。

_registry = {}
class Register:
    def __init__(self, name):
      	self.name = name

    def __call__(self, f):
        _registry[self.name] = f
        return f
    
@Register("a1")
def func(arg):
    print(arg)
